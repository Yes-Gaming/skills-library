### Q7: Immediate Eliminations (Open)
**Which logos would you eliminate immediately? Why?**

**Logo 3 and Logo 6 - Eliminated for functional failures:**
Script typography fails basic usability requirements. Reduced legibility at small sizes (below 48px these become illegible). Slower cognitive processing time - script requires character-by-character recognition rather than word-shape recognition. Poor rendering on lower-resolution displays. Accessibility issues for users with visual impairments. From a practical standpoint, these fail fundamental functionality tests. Engineering principle: don't choose style over function when function is critical.

**Logo 7 - Eliminated for communication failure:**
Monogram creates unnecessary cognitive load. Users shouldn't have to interpret or remember what letters stand for. Functionally inefficient - requires mental translation step (YS = Yes Games) every time you look for the app. During time-critical situations (live betting), that processing delay is a functional defect. Practical assessment: creates problem instead of solving one.

**Logo 4 - Eliminated for over-engineering:**
Bold + shadow is excessive styling that doesn't improve function. Shadow effect adds rendering complexity, increases file size, reduces clarity at certain sizes. It's over-designed - adding elements that don't serve functional purpose. Engineering principle: optimal solution uses minimum necessary components. Logo 4 violates this - the shadow doesn't make it more functional, just heavier. Would work, but inefficient solution.
